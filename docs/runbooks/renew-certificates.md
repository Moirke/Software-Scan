# Runbook: Renewing TLS Certificates

Server certificates generated by `deploy/generate-certs.sh` are valid for
one year.  The CA certificate is valid for 10 years and typically does not
need to be replaced.

---

## Check when the current certificate expires

```bash
openssl x509 -enddate -noout \
    -in /opt/repo-scanner/certs/server.crt
```

Example output:
```
notAfter=Feb 21 17:30:00 2027 GMT
```

Set a calendar reminder ~2 weeks before expiry.

---

## Renew the server certificate (CA stays the same)

Run this on your **workstation** from the root of the repo.  The existing
`deploy/certs/ca.key` and `deploy/certs/ca.crt` must be present — you only
need to re-sign a new server cert.

```bash
bash deploy/generate-certs.sh scanner.corp.example.com
```

The script overwrites `server.crt` and `server.key` while leaving `ca.crt`
and `ca.key` unchanged.

---

## Copy the new cert and key to the VM

```bash
scp deploy/certs/server.crt user@scanner.corp.example.com:/opt/repo-scanner/certs/
scp deploy/certs/server.key user@scanner.corp.example.com:/opt/repo-scanner/certs/
```

---

## Reload Nginx (no downtime)

```bash
ssh user@scanner.corp.example.com
docker compose -f /opt/repo-scanner/docker-compose.yml exec nginx nginx -s reload
```

Nginx reloads the cert in place — there is no interruption to traffic.

---

## Verify

```bash
openssl s_client -connect scanner.corp.example.com:443 \
    -CAfile deploy/certs/ca.crt \
    < /dev/null 2>/dev/null \
    | openssl x509 -noout -dates
```

Confirm `notAfter` shows the new expiry date.

---

## If you also need to replace the CA

This is only needed if the CA key is compromised or the CA cert is expiring
(10-year validity).  Replacing the CA requires re-distributing `ca.crt` to
every caller — browsers, CI agents, curl clients.

1. Delete the old `deploy/certs/` directory
2. Re-run `bash deploy/generate-certs.sh <hostname>` — a fresh CA and server
   cert will be generated
3. Copy new `server.crt` and `server.key` to the VM
4. Reload Nginx
5. Distribute the new `ca.crt` to all callers following the instructions in
   `docs/tutorials/deploying-to-rocky-linux.md` (Step 4)
